# LP ブランドデザイン設定書

**プロダクト**: Eldonia–Nex  
**URL**: https://eldonia-nex.com/  
**バージョン**: v0.4.0（本番デプロイ済み）  
**最終更新**: 2026-06-23  
**地位**: 添付 LP モックアップを **正式ブランドデザイン基準** とし、`/` に実装したランディングページの仕様書

---

## 1. 目的

添付 LP 画像（ダークファンタジー / 月夜の魔法都市）を Web 上で再現し、以降の UI 実装（各モジュール・設定画面等）のビジュアル参照とする。

| 項目 | 内容 |
|------|------|
| フレームワーク | Next.js 16 App Router + TypeScript |
| スタイル | Tailwind CSS v4 + `globals.css` の LP 専用クラス |
| 画像 | `next/image` による最適化 |
| アニメーション | 過度な motion は不使用（表示速度優先） |

## 2. デザイントークン

### 2.1 カラー

| 用途 | 値 | Tailwind / CSS |
|------|-----|----------------|
| 背景（濃紺〜黒） | `#05070a` | `bg-[#05070a]` / `.lp-page` |
| 金（アクセント） | `#c5a059` | 枠線・ボタン・見出し |
| 金（明） | `#e8d5a3` | 本文見出し・ボタン文字 |
| テキスト muted | `#9a8b6a` | 補助文 |
| 紫グロー（Premium） | `rgba(139,92,246,0.35)` | `LpCard glow="purple"` |
| カード背景 | `#0a0e18/75` + `backdrop-blur-sm` | 半透明ガラス風 |

### 2.2 タイポグラフィ

| 用途 | フォント | 備考 |
|------|----------|------|
| 見出し・ナビ・ボタン | **Cinzel**（`font-display`） | `src/app/layout.tsx` で Google Fonts 読込 |
| 本文 | **Noto Serif JP** | 既存 Eldonia ブランドと統一 |

> モックアップ記載の Playfair / Inter ではなく、リポジトリ既存の Cinzel + Noto Serif JP を採用。

### 2.3 ビジュアル要素

- **EN 紋章**: `aset/logo.png` → `/aset/logo.png`
- **金の枠線**: `LpCard` / `LpGoldFrame` / ボーダー `#c5a059` 系
- **ゴシック装飾**: セクションタイトル左右 `✦`、`LpGoldFrame` 上下グラデーションライン
- **紫のグロー**: Premium プランカード・翻訳 A⇄あ バッジ
- **高級ピンバッジ**: `aset/lp/pin-badge.png` → `/aset/lp/pin-badge.png`

## 8. モックアップとの差分（v0.4.0）

要件どおり **雰囲気・構成・必須文言** は反映済み。以下は参照画像との **視覚的差分**（今後の改善候補）。

| 項目 | モックアップ | 現行実装 | 優先度 |
|------|-------------|----------|--------|
| World 装飾枠 | 高度なフィリグリー SVG | CSS グラデーションライン + 簡易枠 | 中 |
| Hero 看板 | 木製吊り看板イラスト | CSS `.lp-sign-board`（PC のみ表示） | 低 |
| FeatureCards アイコン | 金線画アイコン | 絵文字プレースホルダ | 中 |
| Translation 地球儀 | 3D 風ゴールド地球 | `aset/lp/globe.png` 使用 | — |
| Services アイコン | 統一金線画 | `aset/*_icon_1024.png` | — |
| Lab アイコン | フラスコ線画 | `lab_icon_1024.png` | — |

## 11. 文言・デザイン変更手順

1. **コピー変更** → `src/lib/lp/content.ts` を編集
2. **画像差し替え** → `aset/` を更新 → `npm run aset:sync`
3. **色・装飾** → `globals.css` の `.lp-*` または各セクションの Tailwind クラス
4. **検証** → `npm run build` 後、本番または `npm run dev` で `/` を確認

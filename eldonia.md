# Eldonia-Nex ドキュメント要約

`docs`フォルダー内のドキュメントに基づき、Eldonia-Nexプロジェクトの概要、要件、システム構成、および開発ルールを要約します。

---

## 1. プロジェクト概要 (参照: 00, 08)

**Eldonia-Nex** は、すべてのクリエイターが作品を販売やイベントで収益化できる、包括的な統合プラットフォームです。
「クリエイターのための創作プラットフォーム」を副題とし、創作活動を通じた経済的自立と創造性の向上を支援します。

*   **ビジョン**: すべてのクリエイターが自由に表現し、正当な評価と収益を得られる世界の実現
*   **ターゲット**: 個人クリエイター、企業・団体、コンシューマー
*   **バージョン**: 2.0 (段階的アーキテクチャ設計)

---

## 2. 主要機能 (参照: 01, 08)

Eldonia-Nexは以下の主要機能を提供します。

1.  **ギャラリー・作品管理**: 画像、動画、音声、ドキュメントの投稿・販売。AI審査機能付き。
2.  **コミュニティ機能**: グループ作業、チャット、ビデオ会議、ファイル共有。
3.  **イベント機能**: オンライン/オフラインイベントの作成、チケット販売（QRコード対応）。
4.  **ショップ (EC) 機能**: デジタル・物理商品の販売、在庫管理、発送管理。
5.  **Live配信機能**: リアルタイム配信、投げ銭（ギフト）、有料配信。
6.  **仕事・マッチング機能**: クリエーター検索、案件管理、契約支援。
7.  **ゲーミフィケーション**: EXP、レベルシステム、実績バッジ、ランキング。
8.  **サブスクリプション**: Free, Standard, Pro, Businessの4プラン。
9.  **多言語・多通貨対応**:
    *   言語: 日本語、英語、韓国語、中国語、ベトナム語
    *   通貨: JPY, USD, EUR, KRW, CNY, VND
10. **管理機能**: ユーザー管理、コンテンツモデレーション、売上分析。

## 3. システム構成と技術スタック (参照: 06, 09)

Eldonia-Nexは**段階的アーキテクチャ移行**（モノリシック → ハイブリッド → マイクロサービス）を採用しています。

### 技術スタック (最新版)
*   **フロントエンド**:
    *   Next.js 15.0+ (App Router), React 18.3+, TypeScript 5.7+
    *   Tailwind CSS 3.4+, Shadcn/UI
*   **バックエンド**:
    *   Django 5.1+, Django REST Framework 3.15+, Python 3.12+
    *   Celery 5.4+ (非同期処理), Django Channels 4.1+ (WebSocket)
*   **データベース・インフラ**:
    *   PostgreSQL 17+, Redis 7.4+
    *   Docker 27.3+, Kubernetes 1.31+
*   **外部連携**: Stripe (決済), SendGrid (メール), OAuth (認証)

### アーキテクチャフェーズ
*   **Phase 1 (現在)**: モノリシック構成。Next.js + Django REST API。
*   **Phase 2**: ハイブリッド構成。ファイル処理や配信機能を分離。
*   **Phase 3**: マイクロサービス構成。機能ごとにサービスを完全分離。

## 4. 開発環境ルール (参照: 11)

開発チーム全体で統一された環境構築と運用のためのルールです。

### プロジェクト構成
*   `frontend/`: Next.js アプリケーション (Port 3000)
*   `backend/`: Django REST API アプリケーション (Port 8000)
    *   **重要**: `manage.py`の実行は必ず `backend/` ディレクトリで行うこと。

### 実行コマンド例
```bash
# Backend (Localhost:8000)
cd backend/
python manage.py runserver 0.0.0.0:8000

# Frontend (Localhost:3000)
cd frontend/
npm run dev
```

### コーディング規約
*   **Frontend**: ESLint + Prettier, コンポーネント名はPascalCase。
*   **Backend**: PEP 8準拠, Black + isort使用。
